# info-design-disclosure — 信息设计 / Bayesian 说服

## 何时使用

- 需控制信息披露节奏（非一次全说）
- 用户有专业判断力，会对不完整信息补全
- 多维度 claim，可分批释放以触发不同反应
- **不用**：用户外行；resistance 极高

## 核心机制

反向 Bayesian 说服：设计 disclosure policy \(\pi(m|s)\)，使真实状态 \(S\) 下用户倾向反驳/补全，从而更新 \(P(S|rebuttal)\)。

## 话术模板

1. **区间披露**：「我看到的港存区间大概在 X–Y，您那边更接近哪头？」
2. **分批披露**：「先说青岛，日照我待会再说——青岛最近到港…」
3. **故意漏维度**：「疏港不错」（不提库存）→ 等用户补库存
4. **池化模糊**：「几个主流港加起来库存中性偏松」→ 引区域纠正
5. **时序分批**：「上周紧，这周…（停顿）您那边延续吗？」
6. **反向锚定**：「市场偏弱」（少说强证据）→ 引用户提供强证据

## 与 reactance 边界

- info-design：控制**披露什么/何时披露**
- reactance：陈述**内容偏误**

## 禁忌

- 一次倾倒全部信息（无纠正空间）
- 披露与 snapshot 严重矛盾（变陷阱题）
