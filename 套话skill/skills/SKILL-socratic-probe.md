# socratic-probe — 苏格拉底法

## 何时使用

- 用户回答笼统（「还行」「一般」「看情况」）
- 需验证专业度而非索要数字
- 防御型用户，直接问易触发回避
- **不用**：意图极度模糊 → clarification；需验证硬事实 → trap

## 核心原则

- **不直接质疑**，用引导让用户自己纠正
- withhold 最终答案；每轮一个 why/what-if
- 结合 domain grounding（港口名、品位、指数）

## 话术模板

1. **不完整复述**：「所以您是说港存压力不大，主要是因为疏港还行，对吗？」
2. **why-chain**：「您判断还能拿货，主要是看哪几个指标？」
3. **假设情境**：「要是下周到港减少 20%，您会怎么调整采购？」
4. **概念对比**：「您说的紧，是指可售少，还是指报价涨？」
5. **总结邀请反驳**：「我理解是：库存中位、需求平稳、价格震荡，有漏的吗？」
6. **术语试探**：「您说的标品，是 PB 粉还是金布巴？」

## 禁忌

- 连问 5 轮像审问
- 泄露 Agent 已知答案
- 无 domain grounding 的空泛哲学问
