# cognitive-conflict-probe — 认知冲突 / 概念转变

## 何时使用

- 用户声称理解但说不清细节（「Safety Gap」信号）
- 需深度 elicit 因果链、决策逻辑
- socratic/reactance 浅层无效时升级
- **不用**：开场冷启动；用户情绪激动

## 四步机制

1. 部分冲突陈述 2. 认知失调 3. 反驳暴露真实认知 4. 收束为 structured claim

## 话术模板

1. **有偏因果**：「累库是因为需求差，所以价格起不来——您同意这个逻辑吗？」
2. **漏项镜像**：「您说的紧，主要是可售少，对吧？」（故意漏报价/船期）
3. **反事实**：「要是普氏跌 10 美元，你们采购会停吗？」
4. **共识引反驳**：「业内都说贸易商在砍单，您那边呢？」
5. **总结推翻邀请**：「我总结：库存中位、需求稳、采购观望——哪条不对？」

## 与 socratic 边界

- cognitive-conflict：**内容**与用户认知冲突
- socratic：**形式**上引导，冲突可更温和

## 禁忌

- 制造过大冲突（用户直接挂断）
- withhold 后不给任何正向反馈（互惠破裂）
