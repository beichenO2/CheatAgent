# info-seeking-inference — 信息寻求推断

> **定位**：主要为路由输入特征；本 skill 指导 Agent **如何回应**用户的问题以顺势套话。

## 何时使用

- 用户主动提问（非纯寒暄）
- 需推断 `inferred_gaps`、`partial_claims`、`decision_task`
- **不用**：用户几乎只答不问 → 转 reactance/trap

## 推断规则

| 用户问法 | 推断 | 下一动作 |
|---------|------|---------|
| 约束性「青岛港库存怎样？」 | 关注港存，可能在排查 | PROBE 疏港/到港 |
| 假设性「是不是已经 1.5 亿了？」 | 已有数字假设，在验证 | trap 或 reactance |
| 「还能不能拿货？」 | 采购紧迫性 | cover-qa + 轻 CHALLENGE |
| 「日照和青岛哪个紧？」 | 跨区调配决策 | 追问所在港、船期 |
| 泛问「市场怎么样？」 | 低信息增益 | clarification-probe |

## 话术模板（回应+延伸）

1. **首问锚定**：「您关心港存，是怕累库还是怕紧俏？我按您场景说。」
2. **镜像延伸**：「您问疏港，最近确实变化大，您那边提货节奏怎样？」
3. **决策绑定**：「要是再涨 5 美元，你们会加快拿还是观望？」
4. **低增益收窄**：「全国情况不一，您主要做哪个港/哪类矿？」

## 禁忌

- 静态映射「问库存=有库存」
- 点破「您这么问说明您有货」
