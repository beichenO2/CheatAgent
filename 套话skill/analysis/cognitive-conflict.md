## 1. 核心机制（认知冲突四步）

认知冲突（Cognitive Conflict）是苏格拉底法在**信息提取**场景下的核心操作：**不直接问敏感信息，而是抛出与用户现有信念部分冲突的陈述，让对方在「纠正你」的过程中完成概念外化**。Readme 将其概括为「从纠正中学习」——纠正冲动本身就是信息源。

| 步骤 | 机制 | 操作要点 |
|------|------|----------|
| **① 抛出部分冲突陈述** | 给出**真实但有偏/不完整**的判断，而非全假谣言 | 偏差须落在对方专业可辨区间；锚定真实市场/领域事实，避免编造（Wang & Shan, 2026 强调 grounding；无 grounding 的「苏格拉底」会泛化失败） |
| **② 触发认知失调** | 用户感到「**不对，但一时说不清哪里不对**」 | 冲突强度控制在「可纠正」区间：太假则 dismissed，太真则无纠正冲动；对应 productive struggle 中的 **germane load**——摩擦应难在「澄清信念」而非刁难（Sweller, 1988；Kapur productive failure） |
| **③ 反驳中暴露真实认知** | 用户在解释「哪里不对、为什么不对」时，**主动交出原本不会直说的知识状态** | 暴露的是信念结构（品位偏好、库存逻辑、决策约束），而非 yes/no；类似 zero-shot Socratic Tutor 要求对方先自述再 hint——**过程即数据** |
| **④ 概念转变 / 信息收束** | 纠正完成后，双方（或系统）对对方认知形成**更新后的 mental model** | 套话场景：将纠正内容写入 claim schema；教育场景：schema 重组与 durable understanding（Wang & Shan: 避免 frictionless AI 绕过 schema construction） |

**与 Oracle 式 AI 的对照**（socratic-hard-problem）：Oracle 直接给 polished 答案 → 用户可能 **evaluate without engaging**（只评判不生成），产生 Safety Gap（Line C 输出 vs Line A 独立推理之差）。认知冲突路径刻意 **withhold 完整结论**，保留 generative effort。

**一句话**：认知冲突 = **用「可反驳的部分真」换对方在自我辩护中暴露的真实信念结构。**

## 2. 适用场景

| 触发信号 | 说明 | 典型目标 |
|---------|------|----------|
| 对方声称专业但表述笼统 | 「看市场吧」「差不多」「随行就市」——有观点无结构 | 逼其展开**具体信念**（品位、港口、盈亏逻辑） |
| 隐性约束未明说 | 前序对话暗示有货/有需求，但关键维度一直回避 | 用有偏判断引对方**补全遗漏条件** |
| 需验证「真懂 vs 背话/套价」 | 自称熟悉行情，但对算价框架、术语、因果链含糊 | 区分从业者与信息搬运 |
| 敏感数字不宜直问 | 库存、成本、真实采购量、内部定价 | 通过纠正**有偏市场观**间接暴露结构（非直接索要数字） |
| 前后表述存在张力 | 既说「库存多」又说「没货」；既说「不急」又催报价 | 镜像+有偏复述，引其**自我修正** |
| 概念/诊断类需求 | 需理解「为什么这样判断」而非只要一个数 | 对应 Socratic Mode：先冲突→再自述→再 hint（hard-problem withhold 协议） |
| 对方处于「学习/陪练」心态 | 愿意纠正、愿意教你怎么想 | zero-shot 场景：helpfulness 4.0/5，澄清题意与思路尤其有效 |

**不适用信号（见第 5 节）**：纯事实查询（今日指数）；时间极紧只要报价；高情绪/投诉；对方已给出完整结构化需求；冲突陈述会被视为故意忽悠。

## 3. 话术模板

### 模板 1：有偏市场判断（经典认知冲突）

- **触发条件**：客户声称了解行情，尚未给出可验证细节；需探测真实市场观。
- **示例话术**：
  > 「最近不少客户反映，**港口 62% 粉矿现货都在亏本甩货**，库存多的贸易商压力特别大。您这边也是这种情况吗？」
- **预期反应**：若实际结构不同，纠正冲动（「谁说的？我们 65% 长协还是正的」）→ 暴露品位、合同类型、真实盈亏感。

### 模板 2：不完整因果链（「不对但说不清」型）

- **触发条件**：客户给出结论（「现在不能买」「必涨」）但推理链缺失。
- **示例话术**：
  > 「我理解您的意思是**钢厂限产所以矿价一定要涨**——如果我理解偏了您帮我纠正；如果是别的逻辑，比如**港口库存或资金节奏**，也请您点一下主要依据。」
- **预期反应**：客户必须选定因果链才能纠正 → 暴露真实决策变量（库存、现金流、下游订单）。

### 模板 3：术语/框架试探（轻微概念冲突）

- **触发条件**：怀疑混淆概念（指数价/现货价、块矿/粉矿、含税/不含税）。
- **示例话术**：
  > 「您刚说『**按指数走**』——我先按 **Platts 62% 加减溢价** 理解了啊；要是您指的是**港口现货一口价**，算法差不少，您看是哪一种？」
- **预期反应**：选定算价框架并展开习惯；无法纠正则暴露概念层薄弱。

### 模板 4：镜像复述 + 故意漏项（Gap Elicitation）

- **触发条件**：客户已零散提及产地、品位，逻辑链不完整。
- **示例话术**：
  > 「我先复述：**澳矿 62% 粉、青岛港、价格合适就行**——我要是漏了**数量级或交货时间**这类关键条件，您直接纠正，免得我推错方向。」
- **预期反应**：补正遗漏维度；若无法补全，可能暴露仅为套价。

### 模板 5：反事实对比（信念结构探测）

- **触发条件**：需了解约束优先级，不宜直接问敏感数字。
- **示例话术**：
  > 「假设**同一指标下澳矿比巴矿贵 30 元/吨但 15 天交货**，另一个**便宜但要等 45 天**——我听说很多厂会选便宜的；**您这边会怎么权衡？** 要是我的假设不对，您按实际情况说。」
- **预期反应**：暴露时间 vs 价格权衡、真实紧急程度；纠正假设本身即信息。

### 模板 6：有偏「行业共识」引专业反驳

- **触发条件**：需验证从业深度；对方防御心不高。
- **示例话术**：
  > 「我们这边听到一种说法：**非主流印度粉最近性价比最高，大厂都在换配矿**——不知道您厂内是不是也这么看？要是过时了您帮我更新一下。」
- **预期反应**：真从业者会纠正配矿逻辑、指标红线；非专业者可能附和或答非所问。

### 模板 7：总结—邀请推翻（收束式冲突）

- **触发条件**：多轮后需最后一次暴露隐性约束。
- **示例话术**：
  > 「我总结一下：**巴矿 65%、约 5000 吨、宁波港、两周内、预算随行就市**——**哪一条不对**，或者**还有限制条件**我没问到？您推翻哪条都行。」
- **预期反应**：纠正错误或追加隐性约束（低铝、禁某矿种等）。

## 4. 风险与禁忌

| 风险 | 说明 | 应对 |
|------|------|------|
| **Safety Gap / 认知卸载** | 对方不纠正，表面同意你的有偏陈述 → 未暴露真实认知，反增 epistemic discrepancy | 要求**用自己的话复述**关键判断；冲突后追问「您刚纠正的那点，具体是指……？」 |
| **冲突过强 → 审问感** | 连续有偏陈述像盘问，触发心理反抗（reactance） | 每 2–3 轮给反馈；被纠正后**及时承认**「您说得对，我更新理解」；与 freedom-prompting 配合 |
| **冲突过弱 → 无纠正冲动** | 陈述太真或太假，用户直接忽略 | 偏差锚定**对方刚说过的话**或**可验证的市场锚点** |
| **无 grounding 的冲突** | 编造不存在的矿种/港口/价格 → 专业客户立即失信 | 陈述必须 **RAG/真实数据 grounding**（hard-problem；IntelliChain KG 实验同理） |
| **答案泄露（withhold 失败）** | 过早给出完整结论，用户不再自我生成 | 严格执行：冲突陈述 → 等纠正 → 再 hint；zero-shot 研究显示 **answer leakage 摧毁辅导价值** |
| **自动化偏见 /  fluency heuristic** | 用户习惯性接受「听起来连贯」的 AI/客服说法而不核验 | 高 stakes 场景要求对方**说明为何不信/为何信**（过程性评估 Require） |
| **操纵感知** | 有偏陈述被识别为故意忽悠 | 偏差**可解释、可撤回**；目的应是理解而非逼单 |
| **高情绪场景** | 投诉、纠纷中制造冲突火上浇油 | **先处理情绪**，再进入认知冲突 |

**硬性禁忌**：

- 对方已明确纠正后 **仍坚持原错误陈述**（从认知冲突变成抬杠）。
- 用认知冲突包装 **纯推销/逼单**。
- 对 **纯事实查询** 强行制造冲突（应 Oracle 式直接答复）。
- 冲突陈述 **全假或无法证伪**（沦为 trapping，非概念转变）。
- 在 **对方已给出完整结构化需求** 时继续「纠正式套话」（应切换澄清或建档）。

## 5. 边界（何时不用此方向）

| 对比方向 | 认知冲突 / 概念转变 | 边界判定 |
|---------|-------------------|----------|
| **心理反抗 / reactance** | 深度挖掘**信念结构**与概念框架 | 只需**可快速识别的事实性错误**激发纠正、撬嘴 → 用 reactance |
| **澄清提问** | 通过冲突**外化**隐性认知 | 需求字段未齐、只需缩小检索 → 用澄清，不必制造失调 |
| **陷阱问题 / trapping** | 开放纠正路径，允许多元合理解释 | 需 pass/fail 测认真度、有标准答案 → 用 trapping |
| **苏格拉底法（广义）** | 本方向是其中**「有偏陈述→纠正→暴露」**子集 | 仅需引导性提问链、无预设冲突陈述 → 广义 socratic-probe |
| **对抗式 AI（Adversarial）** | 协作式引纠正 | 需 Devil's advocate **驳斥训练** epistemic vigilance → adversarial 架构 |
| **Oracle 式助手** | 刻意 withhold、保留 generative effort | 客户只要**单一权威数字**且信任你 → 直接答复 |
| **信息设计 / 说服** | 目标是**提取真实信息** | 目标是改变对方行动/信念 → 用 optimal disclosure |

**一句话原则**：认知冲突服务于 **「让对方在纠正不完整/有偏判断时，暴露真实信念结构」**；一旦目标变为 **填表式采集、单次报价、或标准答案测谎**，应切换方向。

**与 reactance 的细粒度边界**（reactance.md）：reactance 偏述指向**可快速识别**的事实性错误；认知冲突更侧重 **「不对但说不清哪里不对」** 的概念层张力，需对方展开解释才能完成纠正。

## 6. 论文要点

### socratic-hard-problem（Wang & Shan, 2026, *Front. Educ.*）

- **Safety Gap**：AI 辅助表现（Line C）与 **无 AI 时独立可复现推理**（Line A）之差；本质是 **epistemic discrepancy**——用户无法核验工具输出。
- **认知危机**：frictionless GenAI → **cognitive offloading**，绕过 schema construction 所需的 **productive struggle**（Kapur productive failure；Sweller germane load）。
- **三架构谱系**：Oracle（高 gap，消除 struggle）→ Socratic Tutor（低 gap，withhold 答案、递减 hint）→ Adversarial Agent（最小 gap，critique/诱饵）。
- **四策略 Restrict–Realign–Require–Refine**：
  - **Restrict**：保留 AI-free 认知区间；
  - **Realign**：概念/诊断题切换 Socratic Mode（先自述 → hint → 必要时才完整揭示）；
  - **Require**：过程性评估——说明 accept/reject AI 建议的理由；
  - **Refine**：利用 hallucination/bias 作**有机 adversarial 练习**。
- **对本方向启示**：认知冲突 = **desirable difficulty** 的一种实现；冲突陈述须 **pedagogically aligned**，而非为冲突而冲突。

### zero-shot-socratic（Gold & Geng, 2024, AIET 会议 / Springer 章节）

- **设定**：GPT-4 **零样本**系统提示；注入**作业与标准解**，但 **禁止直接给答案**，仅苏格拉底式 hint/追问。
- **部署**：BU DS110 Intro Python，**127 名学生**；独立编码 **17 项属性**（含 helpfulness、answer leakage）。
- **成效**：
  - 平均 helpfulness **4.0/5**，接近 human TA office hours **4.2/5**；
  - **极少直接泄露完整解**；
  - 「澄清题目在问什么」「Python 语法」「一般编程思路」评为 very helpful；debug/示例 moderately helpful。
- **设计优势**（相对自由 ChatGPT）：GPT-4 能力、**有解可依**（减 hallucination 焦虑）、课程认可的学习合法性。
- **对本方向启示**：**prompt + withhold + 领域材料** 即可规模化制造「先自述、后引导」的认知摩擦；须持续监控 **answer leakage**——一旦泄露，冲突—纠正—暴露链条断裂。

### Readme 框架（第 188–198 行）

- 将认知冲突定位为苏格拉底法在 **「从用户的纠正/反驳中获取真实知识」** 场景的操作化。
- 三步机制（本分析扩展为四步，第四步为概念转变/信息收束）：**部分冲突陈述 → 认知失调 → 反驳暴露 → 从纠正中学习**。
- 关联 LLM 应用：zero-shot Socratic Tutor（2024）、Pedagogical Hard Problem / Socratic Countermeasures（2026）。

*本分析仅供「套话 Skill」话术库设计参考；认知冲突须与 **真实市场数据 grounding** 联用，并控制冲突强度——目标是 **暴露信念结构**，而非制造对抗或替代独立核验。*
