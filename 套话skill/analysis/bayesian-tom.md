# 贝叶斯心智理论 — 研究方向分析

> **方向 ID**：`bayesian-tom`  
> **研究来源**：Rothe, Lake & Gureckis (2018) *Do People Ask Good Questions?*  
> **与 info-seeking 区分**：本方向侧重 **Bayesian ideal observer / 心理状态后验推断**，而非仅从提问字面推断信息缺口。

---

## 1. 核心机制

### 1.1 Bayesian Theory of Mind

将用户视为在隐藏状态空间 \(H\) 上做推理的 agent。可观测行为（提问、反驳、沉默、语气词）是 likelihood 信号，Agent 维护后验 \(P(h \mid \text{observations})\)。

Rothe et al. 的 ideal observer 在大型状态空间上计算 EIG/ES，选出**期望效用最大**的问题。套话 Agent 的镜像任务是：

```
观测用户行为 → 更新 P(头寸方向, 库存水平, 采购节奏, 专业深度)
             → 选择最小成本 elicitation 动作
```

### 1.2 生成瓶颈 vs 评估能力

关键发现：人**很少**自发提出最优问题，但**能准确评估**他人提出的好问题。  
套话含义：

- 不要期待用户主动给全信息
- 可以抛出「半对半错」的候选问题/陈述，观察用户**接受/纠正/忽略**——纠正强度是强信号
- Agent 侧应优先设计 **high-ES**（绑定决策）的 elicitation，而非泛问

### 1.3 心理状态层

| 层 | 推断对象 | 观测信号 |
|----|---------|---------|
| 认知层 | 知道什么数字/指标 | 纠正精度、术语使用 |
| 动机层 | 想涨还是想跌、想出货还是补库 | 问题方向、反驳语气 |
| 元认知层 | 是否意识到自己在暴露信息 | 犹豫、转移话题、反问 Agent |

---

## 2. 适用场景

- 用户行为序列 ≥2 轮，有足够 likelihood 累积
- 需判断「用户是真不懂还是在装懂/隐瞒」
- 需在多种话术方向间做**概率化选择**（非硬规则）
- 铁矿石域：结合已知身份（厂长≈多头）作先验，用行为更新

**不适用**：冷启动首轮、用户完全不回应——后验仍接近先验。

---

## 3. 话术模板

### 模板 1：候选假设抛球

「我听说青岛港库存已经破 1.4 亿了，你们那边也是这个量级吗？」  
→ 观察：直接纠正 / 模糊认可 / 转移话题 → 更新 `inventory_belief` 精度

### 模板 2：决策绑定试探

「要是下周矿价再涨 5 美元，你们会加快拿货还是观望？」  
→ 观察：选项偏好 → 更新 `procurement_urgency` 与头寸方向

### 模板 3：专业度分层问

「你们现在更看普氏还是港口现货成交？」  
→ 观察：回答深度 → 更新 `expertise_level`

### 模板 4：二选一后验探测

「最近是疏港快还是到港快？我这边数据有点打架。」  
→ 观察：选边 + 补充细节 → 更新 `logistics_state`

### 模板 5：沉默压力（低侵入）

「嗯，那这块您方便多说一点吗？我想确认下是不是和我看到的情况一致。」  
→ 观察：补充意愿 → 更新 `resistance_level` 与 `cooperation_prior`

---

## 4. 风险与禁忌

- 禁止在后验未收敛时连发 3 次以上 CHALLENGE
- 禁止把单次模糊回答当作高置信推断
- 先验权重过高（仅凭「厂长=多头」）会压制真实信号——需随轮次衰减

---

## 5. 与其他方向边界

| 方向 | 分工 |
|------|------|
| info-seeking | 从「问了什么」推断缺口；本方向从「行为+反应」推断心理状态 |
| reactance | 主动制造偏误激发反驳；本方向可评估反驳强度并入后验 |
| implicit-modeling | 特征工程与意图层；本方向提供概率化决策框架 |
| clarification | 降低 Agent 侧不确定性；本方向降低对用户隐藏状态的不确定性 |

---

## 6. 关键论文要点

- **Rothe et al. 2018**：自然语言 twenty-questions；EIG vs ES；人类评估 > 生成
- **理想观察者**：大状态空间下最优 question 可计算，可作套话策略 upper bound
- **实践启示**：Agent 应用 ES 型 elicitation；用用户「纠正/评估」反应作强 likelihood
