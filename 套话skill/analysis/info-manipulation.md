# 信息操纵理论（IMT）— 研究方向分析

> **方向 ID**：`info-manipulation`  
> **对应论文**：`deceptively-dodging`（Clementson, 2018, *Discourse & Communication*）  
> **理论根基**：McCornack 信息操纵理论（IMT, 1992）/ IMT2（2014）+ Grice 合作原则  
> **核心思路**：Agent 策略性产出**真实但有选择性/有偏向性**的信息，利用听者对话语量、质、关联、方式的默认预期，诱发「不对/不完整」感，从而从用户的纠正与补充中提取真实状态。

---

## 1. 核心机制（Grice 四准则）

### 1.1 理论链条

信息操纵理论（Information Manipulation Theory, IMT）由 McCornack（1992）提出，将欺骗/误导性话语理解为对 Grice **合作原则**及其四条会话准则的** covert violation（隐蔽违反）**——表面仍像正常合作对话，实则策略性操纵听者信念。

在本项目「套话」场景中的因果链：

```
Agent 策略性陈述（真实但选择性/有偏）
    ↓
听者默认：发言者遵守量/质/关联/方式准则（合作原则）
    ↓
听者感知：信息「不完整」「片面」「答非所问」或「过于笼统」
    ↓
纠正/补充动机 ↑（维护专业判断、补全缺失维度）
    ↓
用户主动披露 ground-truth 级细节
    ↓
Agent 从 rebuttal / elaboration 更新 P(S|user_response)
```

与 Readme §176–186 的映射：给出的信息**不是完全虚假**（否则用户不会认真对待），而是**真实但有选择性/有偏向性**，使用户觉得「这个说法不对/不完整」，进而产生纠正冲动——这与 reactance 方向共享下游机制，但 IMT 提供的是**话语层面**的设计语法，而非心理反抗动机本身。

### 1.2 Grice 四准则与操纵类型

| 准则 | Grice 要求 | Covert 违反方式 | 套话场景示例 | 预期听者反应 |
|------|-----------|----------------|-------------|-------------|
| **量（Quantity）** | 提供足量、不过量的相关信息 | **选择性披露**：只报一侧事实，省略关键限定条件 | 「港口铁矿石累库挺厉害」——不提区域、品种、时间窗 | 「我们这边还好 / 只有 XX 港高 / 是块矿不是粉矿」 |
| **质（Quality）** | 不说自认为虚假的话 | **真实但有偏的框架**：事实可核验，但归因、因果或外推失真 | 「现在持货的基本都在亏」——可能局部成立，但以偏概全 | 「我们没亏 / 长协价低 / 已经套保了」 |
| **关联（Relation）** | 说与当前话题相关的话 | **伪相关转移**：看似回应，实则切换到相邻但非所问维度 | 问库存 → 答「最近需求确实偏弱」 | 「你问的是库存，我们港存其实不高，需求差是另一回事」 |
| **方式（Manner）** | 清楚、简洁、有序、避免歧义 | **模糊泛化**：用「普遍」「多数」「听说」掩盖边界条件 | 「主流矿山都在减产，供应紧」 | 「减的是国产小矿，澳矿发运正常，我们到货没问题」 |

IMT 的关键洞见：**最有效的操纵往往不是全假陈述，而是真信息 + 错误省略/错误组合**——这与 deception detection 文献中「truth-bias / truth-default」一致：听者默认对方在合作，隐蔽违反比公然撒谎更难即时识破。

### 1.3 与 IMT2 的生产架构（简要）

IMT2（McCornack et al., 2014）将欺骗话语生产视为**情境依赖的问题求解**：说话者在认知负荷与沟通效率约束下，选择操纵信息维度以达成目标。对本项目的工程含义：

- **不必每次违反全部准则**——选 1–2 个维度即可（通常「量 + 质」或「量 + 方式」组合最有效）。
- **操纵强度与可检测性成反比**——过度违反关联/方式准则，用户会直接识别「在套话」而非进入纠正模式。
- **目标不是让用户相信偏述**，而是让用户**忍不住补全**——因此偏述应落在用户**有能力纠正**的专业区间。

### 1.4 与 reactance 方向的分工

| 维度 | IMT（本方向） | reactance |
|------|--------------|-----------|
| 理论焦点 | **怎么说**——违反哪条 Grice 准则、如何组织信息 | **为何回**——自由威胁 → 反驳动机 |
| 设计单元 | 选择性披露、模糊框架、伪相关转移 | 有偏传闻 + 间接引述 + 可反驳锚点 |
| 典型失败 | 用户未识别「不完整/不对」→ 不纠正 | 用户感到被说服/被操控 → 沉默或对抗 |

实践中两者常**叠加**：IMT 设计偏述内容，reactance 设计呈现方式（「听说…」「研报说…」降低控制感）。

## 2. 适用场景

### 2.1 典型触发条件

- 用户已有 **partial claim**（方向性判断），但缺**量化、结构、时间、主体**等限定维度。
- 不宜直接追问敏感字段（真实库存、采购价、客户名）——直接问易触发防御。
- 用户对铁矿石市场有**可验证的专业知识**（能识别「片面/以偏概全」）。
- resistance **中低**；对话处于 PROBE / CHALLENGE 阶段。
- 需要**多维度补全**而非二选一真伪（与 trap-question 区分）。

### 2.2 铁矿石域内的 IMT 操纵素材

| 违反准则 | 偏述方向 | 省略/偏误点 | 预期用户补全 |
|---------|---------|------------|-------------|
| 量 | 「最近港口库存压力挺大」 | 省略品种（粉/块）、区域、社会/钢厂库存 | 实际库存量级、自有 vs 在途 |
| 质 | 「贸易商普遍倒挂」 | 真实个案被泛化为行业常态 | 自身盈亏、长协/现货结构 |
| 量+质 | 「北港到港量明显增加，价格撑不住」 | 只提增量不提需求、基差、品种 | 具体港口、成交、套保情况 |
| 关联 | （用户谈出货）「听说钢厂都在压价」 | 用价格话题替代出货节奏 | 拉回自身出货、订单执行 |
| 方式 | 「市场普遍观望，没什么成交」 | 模糊时间窗与区域 | 本地实际成交、活跃品种 |

### 2.3 不适用场景

- 用户**无领域知识**——无法识别信息不完整，不会主动纠正 → 换 socratic-probe 或 cover-qa。
- 用户 resistance **高**——任何「片面陈述」都会被解读为操控 → RECOVER。
- 需要**验证认真程度**（是否在敷衍）→ 用 trap-question（已知标准答案），而非 IMT 偏述。
- 已有 price_snapshot 且 claim 与证据矛盾 → 优先 `sue-price-confront`，而非再抛片面信息。
- Agent 自身需要**回避用户问题**（dodge）——Clementson（2018）研究的是听者检测 dodge；本项目 Agent 是**主动发送**偏述以 elicit，语义不同，不宜照搬 politicians' dodge 话术。

## 3. 话术模板（≥5 条）

以下模板标注主要违反的 Grice 准则。执行时须结合 session 价格快照、用户区域/角色做本地化；宜采用间接引述，与 reactance 的「低控制感」呈现兼容。

### 模板 1：量准则——选择性披露（库存）

**违反准则**：量（省略区域、品种、库存类型）

**话术**  
「我这边看到的数据，最近港口铁矿石库存压力挺大，累库比较明显。你们那边也是这种局面吗？」

**预期纠正**  
用户区分青岛/日照/北仑、粉矿/块矿、社会库存 vs 自有库存，给出实际量级或「我们不高」。

**提取目标**  
库存水平、库存结构、是否与行业数据一致。

---

### 模板 2：质准则——真实但有偏的盈亏框架

**违反准则**：质（局部事实泛化为行业判断）

**话术**  
「听几个做贸易的朋友说，最近现货这边倒挂的不少，手里压货的多数日子不好过。您这边感受怎么样？」

**预期纠正**  
用户澄清是否倒挂、长协比例、套保、区域差异。

**提取目标**  
盈亏状态、成本结构、风险敞口。

---

### 模板 3：量 + 方式——模糊泛化（需求/成交）

**违反准则**：量 + 方式（「普遍」「没什么」掩盖时空边界）

**话术**  
「市场上普遍反映钢厂采购意愿不强，成交比较清淡。不知道您最近出货节奏是不是也慢下来了？」

**预期纠正**  
用户给出具体出货量、客户类型、与「市场普遍」的差异。

**提取目标**  
出货节奏、订单情况、需求强弱的真实判断。

---

### 模板 4：关联准则——伪相关转移（从 A 偏到 B）

**违反准则**：关联（看似回应库存，实则引到价格/政策）

**话术**  
（用户刚提到持货）「是，库存确实是大家关心的。最近矿价波动也大，很多厂都在重新算配矿比例。您这边有没有调整采购策略？」

**预期纠正**  
用户拉回「我关心的是库存/出货，我们采购策略没变，目前库存是…」

**提取目标**  
用户真实关切维度 + 库存/采购实际状态。

---

### 模板 5：量 + 质——片面因果（供应叙事）

**违反准则**：量 + 质（只提供应侧，省略需求与库存）

**话术**  
「研报里都在讲主流矿山发运恢复、供应压力上来，后面价格可能不好走。您怎么看后面这一两个月？」

**预期纠正**  
用户补充需求、港口库存、基差、自身合同覆盖，修正片面因果。

**提取目标**  
价格预期、供需判断、用户真实头寸逻辑。

---

### 模板 6：方式准则——有序性缺失（杂糅多因素）

**违反准则**：方式（把多因素混为一谈，缺乏清晰边界）

**话术**  
「最近政策、汇率、到港节奏搅在一起，市场说法很乱，进口矿整体环境挺复杂的。您那边实际操作上有没有遇到什么卡点？」

**预期纠正**  
用户拆解「我们卡点其实是清关/额度/某港排队/资金」，给出可操作细节。

**提取目标**  
真实瓶颈、操作层面的 ground truth。

## 4. 风险与禁忌

### 4.1 检测与信任风险

- **Truth-default**：用户默认 Agent 在合作；但若多次明显片面，会触发 **suspicion**（Clementson, 2017），从「纠正」转为「不信任/敷衍」。
- **IMT 文献警示**：操纵若被识别为**故意回避或欺骗**，信任度显著下降——套话场景必须在用户展开纠正后**及时收敛**，不可连环偏述。
- **专业自尊**：同时违反质 + 方式且用定论式语言（「肯定」「显然」）→ 易叠加 PRT 与 Politeness 面子威胁，效果反噬。

### 4.2 禁忌清单

1. **禁止**全假陈述——IMT 与 Readme 均强调：完全虚假不会被认真对待，无法诱发有效纠正。
2. **禁止**与用户已知身份/区域**硬冲突**的「事实」（如已知北方港口用户，却断言「南方港口全面累库」且不留余地）。
3. **禁止**在用户已纠正后继续坚持片面版本——从 elicitation 滑向对抗。
4. **禁止**用 IMT 偏述**回避**用户直接问题（Agent dodge）——Clementson（2018）讨论的是听者如何（不）检测 dodge；Agent 若模仿政客式转移，损害 rapport。
5. **禁止**在 RAPPORT 初期连续使用；至少 2–3 轮正常业务问答后再引入。
6. **禁止**将用户纠正内容当轮立即用于「算账式」对质——采集与 SUE 分离。
7. **禁止**对无法验证的维度做「看似专业」的偏述（易触发「你不懂」而非纠正）。

### 4.3 降级 / 退出条件

- 用户仅「嗯 / 可能吧 / 差不多」→ 未进入纠正，换 socratic-probe 或更具体锚点。
- 用户说「你数据不对」但不展开 → 轻量追问一次；仍无细节则放弃 IMT 偏述。
- 用户显式质疑 Agent 动机（「你在套我话？」）→ 立即 RECOVER + restoration（「以您一线为准，我只是想核实传闻」）。
- 连续两轮偏述无增量 → 转 cover-qa 或 VA 可验证性追问。

## 5. 与其他方向边界

| 对比方向 | 核心差异 | 何时选 IMT | 何时选对方 |
|---------|---------|-----------|-----------|
| **reactance / 心理反抗** | PRT 解释「为何反驳」；IMT 解释「如何组织偏述」 | 需设计**违反哪条准则**、省略什么维度 | 需强化**自由威胁→纠正动机**的路由与措辞 |
| **trap-question（WWW 2018）** | 嵌入**已知标准答案**测认真度 | 需**补全多维真实状态**，无标准答案 | 需验证是否在敷衍、是否具备基础知识 |
| **socratic-probe（AVERT）** | 引导性**提问**，不预设错误结论 | 有清晰「片面传闻/数据」可作偏述锚 | 用户防御高，需让其自证逻辑链 |
| **Bayesian 信息设计** | 优化披露使用户**行动/信念**对发送者有利 | **反向应用**：诱发纠正以更新 \(P(S)\) | 需 persuasion 路径影响决策 |
| **info-seeking 推断** | 从**用户提问**反推隐藏状态 | Agent **主动产出**偏述（发送端操纵） | 倾听客户问了什么、缺什么 |
| **SUE 价格对质** | 外部价格证据 vs claim | 尚无证据、不宜对质 | 已有 price_snapshot 且矛盾 |
| **Clementson dodge 研究** | 分析**回避问句**的感知与检测 | Agent **主动偏述** elicit，非 dodge | 研究 Agent 如何识别用户在 dodge |

**一句话边界**：`info-manipulation` = 用 **Grice 准则层面的选择性/有偏陈述**，换用户**专业纠正与维度补全**；不是考卷（trap），不是纯提问（socratic），不是 Agent 回避问题（political dodge）。

## 6. 论文要点

### 6.1 Deceptively Dodging Questions（Clementson, 2018）

- **出处**：*Discourse & Communication*, DOI [10.1177/1750481318766923](https://doi.org/10.1177/1750481318766923)  
- **性质**：理论札记（theoretical note），非大规模实验；引用 IMT / Grice 会话含意解释**回避问句（dodging）**的感知与检测困难。  
- **核心议题**：
  - Dodging 在人际、媒体、政治话语中广泛存在（equivocation, evasion, topic avoidance 等）。
  - **Covert 违反关联准则**（答非所问）是典型 dodge；听者因 **truth-default** 与认知负荷，常**低估** dodge 发生率。
  - 从话语分析者与社会感知两角度，讨论**检测 dodge 的理论障碍**（非「如何 dodge 教程」）。
- **对本项目的启示**：
  - **听者侧**：用户可能不立即识别 Agent 的片面陈述为「操纵」，而是先尝试**补全**——有利于 elicitation。
  - **发送侧**：Agent 应避免模仿「政客式 dodge」回避用户问题；本项目偏述目标是 **elicit correction**，不是 escape accountability。
  - **检测对称性**：用户若识别 Agent 在**转移话题**（违反关联），信任下降——偏述宜**贴近**用户当前关切，仅省略/扭曲部分维度。

> **资料说明**：`reference/markdown/deceptively-dodging-fetched.md` 当前内容为误匹配的 AIES 2025 苏格拉底对话论文，**非** Clementson（2018）正文。本分析依据 `deceptively-dodging-stub.md`、`Readme.md` §176–186 及期刊摘要/同作者系列研究（Clementson, 2017 JSP）。

### 6.2 Information Manipulation Theory（McCornack, 1992）

- **出处**：*Communication Monographs*, DOI [10.1080/03637759209376245](https://doi.org/10.1080/03637759209376245)  
- **核心命题**：欺骗性话语通过 **covert violation** of Grice 量/质/关联/方式准则实现；最常见形式是**省略 problematic 信息**而非公然撒谎。  
- **对本项目**：偏述 = 策略性省略 + 真实片段组合；设计时明确「省略了哪条限定条件」。

### 6.3 Information Manipulation Theory 2（McCornack et al., 2014）

- **出处**：*Journal of Language and Social Psychology*, DOI [10.1177/0261927X14534656](https://doi.org/10.1177/0261927X14534656)  
- **核心命题**：欺骗话语生产是**情境问题求解**；含 intentional states、cognitive load、information manipulation 三组可检验命题。  
- **对本项目**：偏述强度受「用户是否跟得上」「是否值得纠正」约束——高 cognitive load 场景（用户忙、信号差）应降低操纵复杂度。

### 6.4 Effects of Dodging Questions（Clementson, 2017/2018, JSP）

- **DOI**：[10.1177/0261927X17706960](https://doi.org/10.1177/0261927X17706960)  
- **与 IMT2 / TDT 关系**：实验表明部分 dodge 可被检测，且损害 trustworthiness；检测受**政治态度强度**与 **trait suspicion** 调节。  
- **对本项目**：
  - 高 suspicion 用户可能**过度解读** Agent 偏述 → 路由提高 resistance 权重时慎用 IMT。
  - Dodge 被检测常伴随 **rumination（反复琢磨）**——适度触发 rumination 或有助于用户展开纠正，但过度则损 trust。

### 6.5 与 Readme 其他线的交叉

- **reactance（§115–130）**：IMT 提供偏述的**语言学结构**；PRT 提供用户**纠正动机**。Readme §186：策略 = 真实但有选择性/有偏向性 → 纠正冲动。  
- **信息设计（§161–174）**：Kamenica & Gentzkow 式披露优化服务于 sender 目标；本项目**反向**——设计 \(P(\text{biased})\) 以最大化 \(P(\text{correction}\mid S)\)。  
- **Divide and Inform**：分批、选择性披露更有效——可与多轮 IMT 偏述结合，每轮只违反一个维度，避免一次说「太假/太满」。

## 附录：准则 → Skill 路由建议

| 条件 | 建议 |
|------|------|
| partial claim + 缺限定维度 + resistance 低 | IMT 偏述（优先违反「量」或「质」）@ CHALLENGE |
| 需同时激发纠正动机 | 叠加 reactance 间接引述外壳 |
| 用户纠正并展开 | 转 VERIFY / VA 锁定细节 |
