# 澄清提问方向分析

> 依据：`reference/markdown/productagent.md`（ProductAgent, EMNLP 2025 Industry Track）

---

## 1. 核心机制

澄清提问是一种**混合主导（mixed-initiative）**的信息收集策略：当对方只给出粗粒度需求（如品类、方向）时，客服不急于下结论，而是基于已有市场/库存信息，**迭代提出有针对性的追问**，将模糊意图逐步收敛为可检索、可报价、可记录的结构化约束。ProductAgent 的三步闭环——**需求汇总 → 候选检索/统计 → 生成下一轮澄清问题**——证明检索/匹配精度随对话轮次单调提升，关键在于问题必须**锚定真实数据分布**（而非凭空编造选项），且每轮聚焦尚未明确的维度。

---

## 2. 适用场景（电话客服套话：铁矿石市场信息收集）

| 触发信号 | 说明 |
|---------|------|
| 开场意图模糊 | 客户只说「想了解铁矿」「最近什么价」「有没有货」，未说明品位、产地、数量、交割方式 |
| 多品类/多规格并存 | 同一品类下 62% 与 65% 品位、澳矿/巴矿/国产矿等分支众多，需快速缩小范围 |
| 信息不足以报价或建档 | 缺少数量级、交货期、港口、付款/结算方式，无法给出有效反馈 |
| 客户表达不完整但愿意配合 | 对方有采购或调研意愿，能接受选择题式引导，而非直接挂断 |
| 需边问边给反馈 | 每轮澄清后同步给出「当前条件下的参考区间/可选货源」，维持对话动力 |

**不适用信号（见第 5 节）**：客户已给出完整采购清单；对方明确拒绝回答细节；仅需单向播报行情。

## 3. 话术模板

### 模板 1：品类/交易类型澄清

- **触发条件**：客户首次来电，仅提及「铁矿」或「铁矿石」，未区分现货/期货、内贸/外贸、块矿/粉矿。
- **示例话术**：
  > 「您好，方便确认一下，您这次主要是想了解**进口粉矿现货报价**，还是**港口块矿成交情况**，或是**远期合约**方向？我按您的方向帮您查。」
- **预期反应**：客户选定一条主线（如「进口粉矿现货」），后续问题可聚焦品位与港口；若答「都想了解」，则按优先级逐层展开。

### 模板 2：品位/规格澄清（高区分度维度）

- **触发条件**：已确认品类为进口粉矿，但尚未说明 Fe 品位、水分、杂质要求；当前市场同品类下 61%/62%/65% 等规格并存。
- **示例话术**：
  > 「进口粉矿这边常见有 **62% 普氏指数对标**、**61.5% 低铝**、**65% 高品** 几类。您这边采购或询价，主要对标哪一档品位？如果有企业内控指标（比如硅、铝上限），也可以说一下。」
- **预期反应**：客户给出目标品位或「按厂内标准」，客服可将需求写入结构化字段，显著缩小报价范围；若不确定，可补充「一般钢厂高炉配矿用 62% 较多」作轻量引导。

### 模板 3：产地/货源澄清

- **触发条件**：品位大致明确，但产地未说明；不同产地（澳洲主流矿、巴西、非主流、国产精粉）价差与可售量差异大。
- **示例话术**：
  > 「这个品位下，最近成交比较活跃的有 **PB 粉/金布巴**、**巴西卡粉**，也有 **马来/印度非主流**。您有指定产地或矿种吗？还是价格合适、指标达标即可？」
- **预期反应**：客户锁定产地偏好或表示「不限，看性价比」，便于检索对应货源池；若客户有合同长协矿，可能直接报出矿种名。

### 模板 4：数量与交货节奏澄清

- **触发条件**：品类与品位已初步明确，但缺少采购量级、装船/到港时间，无法判断是否匹配现货库存或需报远期。
- **示例话术**：
  > 「想了解下计划量级：是 **一船（约 X 万吨）**、**几千吨现货拼车**，还是 **月度长协补货**？大概 **什么时候需要交货或到港**？我好按您的节奏匹配港口可售资源。」
- **预期反应**：客户给出数量级与时间窗，客服可判断走整船报价、港口现货还是排期资源；数量过小可能转入「拼车/甩货」子流程。

### 模板 5：港口/交割方式澄清

- **触发条件**：内贸现货场景，客户未说明提货港、是否含税、自提/配送；同一标的在不同港口价差明显。
- **示例话术**：
  > 「现货这边要看港口。您方便说下是 **青岛港、日照港、北仑** 哪一侧提货？另外是 **港口现货自提**，还是需要 **送到厂**？我按交割方式帮您算到位价。」
- **预期反应**：客户明确港口与交割条款，需求可转化为可执行的 spot 查询条件；若客户在外地，可能转为 FOB/CIF 或就近港转运讨论。

### 模板 6：预算/价格锚点澄清（中后期细化为宜）

- **触发条件**：前述核心维度已基本齐全，仍有多档可售资源；需了解客户价格敏感度或内部预算，避免无效推荐。
- **示例话术**：
  > 「目前符合您指标的资源，港口报价大概在 **X–Y 元/湿吨** 区间。您这边有 **目标采购价** 或 **可接受的上限** 吗？我可以优先帮您筛性价比更高的标的。」
- **预期反应**：客户给出心理价位或「按市场价」，客服可排序推荐；若价格预期严重偏离市场，转入预期管理或替代规格建议。

### 模板 7：用途/配矿场景澄清（提升问题效用）

- **触发条件**：客户对品位表述含糊（「中等品位就行」），或采购用途不明；ProductAgent 表明「适用场景」类维度提问效用较高。
- **示例话术**：
  > 「想再确认下用途：是 **高炉入炉配矿**、**烧结用粉**，还是 **球团原料**？不同用途对硅铝、粒度要求不一样，我问这个是为了避免给您推错货。」
- **预期反应**：客户说明应用场景，客服可反向推导合理指标区间，提高首轮推荐命中率，减少反复澄清。

## 4. 风险与禁忌

| 风险 | 说明 | 应对 |
|------|------|------|
| **问题脱离真实市场** | 凭空编造不存在的矿种、港口或价格，损害可信度 | 选项必须来自当前可售货源/成交统计；不确定时明说「帮您核实」 |
| **重复追问** | 多轮对话中语义重叠（ProductAgent 实证：轮次越多问题相似度越高） | 维护对话记忆，已确认维度不再问；换角度补缺口 |
| **过早细问** | 第一轮就追问杂质、微量元素，客户尚未建立信任 | 先宽后窄：品类 → 品位/产地 → 数量/港口 → 价格/条款 |
| **一次问太多** | 单轮抛出 5 个以上开放问题，电话场景负担过重 | 每轮 2–3 个封闭式/选择题为主，保留「其他/您补充」 |
| **只问不给反馈** | 连续盘问像审问，客户耐心下降 | 每轮澄清后给一句「当前条件下的大致情况」，形成闭环 |
| **诱导性选项** | 为推销特定货源扭曲选项分布 | 选项应反映市场真实分布，不偏袒单一卖方利益 |
| **隐私与合规** | 过度打探对手价格、内部配方、未公开采购计划 | 只收集完成服务所需的最小信息；敏感问题换委婉表述并说明用途 |
| **忽视自由回答** | 客户答「其他」或口述非标需求时强行归入选项 | 记录原文，标注为自定义约束，不丢失信息 |

**硬性禁忌**：在客户已明确拒绝透露某信息后继续追问；用澄清问题做无关营销铺垫；将澄清等同于拖延报价（无反馈的纯提问）。

## 5. 与其他方向的边界（何时不用此方向）

| 对比方向 | 澄清提问 | 边界判定 |
|---------|---------|---------|
| **单向行情播报** | 迭代追问、共建需求画像 | 客户只要「今天 62% 指数多少」且无意向 → 直接播报，不澄清 |
| **苏格拉底式追问** | 目的为快速缩小检索空间 | 客户质疑自身采购逻辑、需反思决策时 → 不用商务澄清，改用辅导式提问 |
| **欺骗/回避检测** | 默认对方真诚但信息不全 | 发现前后矛盾、故意隐瞒 → 转入核验/信任修复流程，而非继续礼貌澄清 |
| **说服与成交推进** | 信息收集阶段，中立 | 需求已清晰、进入议价/逼单 → 停止澄清，切换报价/异议处理话术 |
| **陷阱式测谎** | 开放式、可验证选项 | 需设局验证对方真伪时 → 不用本方向（参见 crowdsourcing-trapping 等） |
| **礼貌/面子维护** | 效率优先、简短直接 | 对方明显感到被冒犯或等级敏感 → 先铺垫关系，降低提问密度 |

**一句话原则**：澄清提问服务于**「把模糊需求变成可执行查询」**；一旦需求已结构化或客户只求一次性事实，应立即停止追问。

## 6. 关键论文要点

- **任务定义**：Product Demand Clarification——从模糊品类出发，通过多轮澄清逐步精化用户意图并检索匹配项（式 1：\( D = \{U_1, A_1, P_1, U_2, A_2, P_2, \cdots\} \)）。
- **架构**：LLM + 工具链（Text2SQL、Category Analyze、Query Generation、Retriever、Question Generation）+ 结构化对话记忆；**问题生成必须 grounded 于目录/市场统计**，避免幻觉。
- **三轮循环**：① 汇总已知约束并分析候选池统计 → ② 检索并展示当前匹配结果 \( P_t \) → ③ 生成 \( n=3 \) 道选择题式澄清问题；用户可选预设项或自由作答。
- **问题设计规律**：适用场景、风格、功能类维度出现频率最高（各 >10%）；系列、规格等过于专业的维度较少被问——**优先问高区分度、客户能答的 facet**。
- **实证结论**：PROCLARE 基准 2000 会话、5 轮澄清后，HIT@10 / MRR@10 **随轮次单调上升**；首轮仅知品类时几乎无法命中目标。
- **统计质量关键**：Question Generation 依赖**当前需求条件下的动态统计**；去掉统计或随机统计可使 MRR@10 下降 3+ 点。
- **模型策略差异**：GPT-4 倾向前期问更宽、后期问更细；GPT-3.5 问题重复率更高但配合 BM25 检索仍有效——提示**问题多样性与检索精度需平衡**。
- **选项来源**：答案选项直接从目录/库存抽取，保证**每个选项至少对应一个真实可售实体**。
- **评估方法**：LLM 用户模拟器（隐藏目标商品）+ 每轮检索指标（MRR@10、HIT@10），支持大规模自动评测。
- **局限**：无真人用户验证；未覆盖价格、评价等关键购买因素；缺乏全局问题规划，后期问题相似度上升。
- **伦理提醒**：澄清能力可被滥用于过度信息收集或推销——电话客服需最小必要原则。
