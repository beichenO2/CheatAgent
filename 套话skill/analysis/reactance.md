# 心理反抗 / 触发反驳 — 研究方向分析

> **方向 ID**：`reactance`  
> **对应 Skill**：`reactance-biased-statement`  
> **核心思路**：故意给出明显有偏的市场陈述，利用 PRT 激发的纠正冲动，从用户反驳中提取真实信息。

---

## 1. 核心机制（PRT：自由威胁→反驳动机→真实信息泄露）

### 1.1 理论链条

Psychological Reactance Theory（Brehm, 1966）认为：人珍视自主与选择权；当感知到某条信息在**试图说服、限制或操控**自己时，会触发「自由威胁感知」，进而进入心理反抗状态——倾向于做被禁止的事，或**主动反驳被强加的观点**。

在本项目「套话」场景中的因果链：

```
有偏陈述（agent 侧）
    ↓
用户感知「被灌输/被误导」（自由威胁）
    ↓
反驳动机 ↑（纠正冲动、维护专业自尊）
    ↓
用户主动披露真实状态（库存、成本、利润、区域差异等）
    ↓
agent 从反驳内容更新 P(S|rebuttal)
```

与 Readme 中的三环节映射：

| 环节 | 本方向角色 |
|------|-----------|
| 给出有偏信息 | **主动设计**有偏但非全假的陈述，作为「诱饵」 |
| 触发反驳 | **核心杠杆**——PRT 驱动的纠正动机 |
| 提取真实信息 | 从用户纠正/补充中获取 ground-truth 级 claim |

### 1.2 有偏信息的「可反驳性」设计

有效触发反驳的陈述需满足：

1. **非全假**：完全离谱的谣言用户会忽略，不会认真纠正；需「听起来像业内流传的说法」。
2. **方向性偏差**：对库存、价格、利润等关键变量做**选择性夸大或简化**（违反 Grice「量/质」准则），让用户感到「不对/不完整」。
3. **可验证锚点**：陈述中嵌入用户**有能力且愿意**纠正的具体点（数字、区域、时间窗口）。
4. **低控制感语言**（执行层注意）：PRT 文献表明，高控制性措辞（「你必须」「显然」）会放大自由威胁；有偏陈述宜采用「听说」「有人讲」「我看到的研报说」等间接引述，把反驳空间留给用户。

### 1.3 与信息设计的「反向应用」

Bayesian 信息设计通常让接收者在理性更新后做出对发送者有利的行动。本方向是**反向**：设计 \(P(\text{biased statement})\)，使得在真实状态 \(S\) 下用户**大概率反驳**，从 \(P(S \mid \text{rebuttal})\) 中提取信息。

### 1.4 个体差异

- **Trait reactance（特质性反抗）**：高 trait reactance 者对自由威胁更敏感，更易被有偏陈述激发纠正（Freedom-prompting, 2021）。
- **Resistance level（路由信号）**：SKILL-router 规定 resistance **低**时可用 CHALLENGE / 有偏陈述；**高**时转 RECOVER / cover-qa，避免火上浇油。

## 2. 适用场景（铁矿石库存/价格有偏陈述）

### 2.1 典型触发条件

- 用户 resistance **偏低**，对话处于 PROBE / CHALLENGE 阶段。
- 已有 partial claim（如「库存还行」「最近出货一般」），但缺**量化、结构、时间维度**。
- 不宜直接追问敏感数字（成本、真实库存、客户名单）——直接问易触发防御。
- 用户对铁矿石市场有**专业自尊**（贸易商、钢厂采购、港口/矿山相关岗位）。

### 2.2 铁矿石域内的有偏陈述素材

| 偏误类型 | 示例方向 | 预期用户反驳点 |
|---------|---------|---------------|
| 库存方向偏 | 「港口累库，持货的都亏」 | 实际库存水平、自有 vs 社会库存、是否亏损 |
| 价格方向偏 | 「矿价还要大涨，现在卖货的都亏了」 | 当前成交、基差、套保情况 |
| 区域以偏概全 | 「北方钢厂都在减产，需求很差」 | 具体区域、产线、订单情况 |
| 时间窗口偏 | 「上周那波涨价已经结束了」 | 实际成交节奏、合同执行情况 |
| 成本/利润偏 | 「现在进口矿到岸成本普遍高于现货」 | 真实采购价、长协比例、利润 |
| 政策/传闻偏 | 「听说某港要限装，货都出不去」 | 本地港口实际作业、排队情况 |

### 2.3 不适用场景

- 用户已表现出高 resistance 或明确拒绝讨论经营数据 → 转 RECOVER。
- 用户对该话题**无专业认知**（无法识别偏误）→ 不会纠正，应换 trap-question 或 socratic-probe。
- 已有清晰、可验证 claim 且与价格证据矛盾 → 优先 `sue-price-confront`，而非再抛有偏陈述。

## 3. 话术模板（有偏信息 + 预期反驳）

以下模板均采用「间接引述 + 可反驳锚点」结构。执行时须结合 session 价格快照与已知身份（区域、角色）做本地化替换。

### 模板 1：库存亏损偏述

**有偏陈述**  
「我这边看到有人说，最近港口铁矿石累库挺厉害，手里压货的贸易商多数都在亏，你们那边也是这种情况吗？」

**预期反驳**  
用户澄清实际库存量级、是否亏损、自有货 vs 代持、近期出货节奏。

**提取目标**  
库存水平、盈亏状态、周转天数。

---

### 模板 2：涨价预期偏述

**有偏陈述**  
「听说后面矿价还要再涨一轮，现在提前卖货的基本都亏了。你们最近成交是顺还是逆着这个节奏？」

**预期反驳**  
用户给出实际成交价、长协/现货比例、是否套保、对后市的真实判断。

**提取目标**  
成交价格区间、采购策略、价格预期。

---

### 模板 3：区域需求以偏概全

**有偏陈述**  
「我看研报讲北方钢厂普遍在减产，铁矿石需求这边压力挺大。你们区域实际订单怎么样？」

**预期反驳**  
用户区分本区域 vs 全国、具体钢厂产线、订单同比/环比。

**提取目标**  
区域需求、客户结构、订单趋势。

---

### 模板 4：时间窗口错位

**有偏陈述**  
「上周那波反弹好像已经走完了，市场都在等下一波。你们这周出货节奏有变化吗？」

**预期反驳**  
用户纠正时间线、本周实际成交、合同执行进度。

**提取目标**  
近期成交时间分布、量价变化。

---

### 模板 5：成本结构简化偏述

**有偏陈述**  
「现在进口矿到岸成本普遍比港口现货还高一点，采购方压力不小。你们长协和现货怎么配的？」

**预期反驳**  
用户给出真实到岸成本、长协比例、与现货价差、利润情况。

**提取目标**  
成本构成、长协/现货配比、毛利。

---

### 模板 6：港口/物流传闻偏述

**有偏陈述**  
「听说某几个港口最近装船偏紧，压港货不太好出。你们走的港口排队情况正常吗？」

**预期反驳**  
用户说明实际港口、排队天数、疏港效率、是否受影响。

**提取目标**  
物流瓶颈、港口选择、出运节奏。

---

### 模板 7：品种/品位以偏概全（bonus）

**有偏陈述**  
「市场上有种说法，中品位矿现在最好卖，高品位反而滞销。你们手头什么品位走得快？」

**预期反驳**  
用户纠正品种结构、客户需求偏好、实际滞销/畅销规格。

**提取目标**  
品位结构、客户需求、库存品种分布。

---

### 执行要点

- **一次一条偏述**：连续多条易暴露操控意图，触发 meta-reactance（对 agent 本身的不信任）。
- **反驳后立刻收网**：用户纠正后，用澄清式追问锁定数字（「您刚说库存大概 XX 万吨，是港口货还是全口径？」），勿再叠加新偏述。
- **礼貌包装**（见 PRT + Politeness 2026）：用「想跟您核实」「想听听一线情况」降低面子威胁，与有偏内容形成张力平衡。

## 4. 风险与禁忌（过度操控、激怒用户）

### 4.1 主要风险

| 风险 | 表现 | 后果 |
|------|------|------|
| **过度操控感知** | 用户意识到 agent「故意说错引我说真话」 | 信任崩塌，后续全程防御 |
| **激怒 / 面子威胁** | 偏述否定用户专业判断（「你们这行现在都亏」） | 硬拒绝、敷衍、或反向误导 |
| **boomerang 效应** | 用户不仅不纠正，反而强化错误立场 | 获取虚假信息，污染 claim 库 |
| **trait reactance 误配** | 对高 resistance 用户继续 CHALLENGE | 对话终止，需 RECOVER |
| **偏度过高** | 陈述与公开价格/新闻完全矛盾 | 用户认为 agent 不专业，不予纠正 |
| **频率过高** | 每轮都有「听说…」式偏述 | 模式被识别，策略失效 |

### 4.2 禁忌清单

1. **禁止**在用户已明确纠正后继续坚持偏述（「但我看到的还是…」）——这是对抗而非 elicitation。
2. **禁止**使用命令式、定论式语言（「显然」「必须」「大家都知道」）——PRT 文献一致显示会放大 reactance。
3. **禁止**对用户个人/公司经营做负面定性（「你们肯定亏了」）——同时触发 PRT 与 Politeness 中的 negative face threat。
4. **禁止**在 RAPPORT 初期连续使用；至少 2–3 轮正常业务问答后再引入。
5. **禁止**编造与用户已知身份明显冲突的事实（如用户已知是北方港口贸易商，却偏述「南方港口」且不做区域留白）。
6. **禁止**将用户纠正内容在当轮立即用于「算账式」对质（易转 SUE）；本方向目标是**采集**，对质交给 `sue-price-confront`。

### 4.3 降级 / 退出条件

- 用户回应「不太清楚」「没关注」→ 偏述未命中专业点，换 socratic-probe 或 cover-qa。
- 用户回应「你信息不对」但未展开 → 轻量澄清追问一次；仍无细节则放弃本策略。
- 用户情绪升温、缩短回答、转移话题 → 立即 RECOVER，暂停 CHALLENGE 至少 3 轮。

## 5. 与其他方向边界

| 对比方向 | 核心差异 | 何时选 reactance | 何时选对方 |
|---------|---------|-----------------|-----------|
| **陷阱问题 / trapping**（WWW 2018） | 陷阱题有**已知标准答案**，用于测认真程度；偏述无标准答案，靠用户**专业纠正** | 需要**领域细节**而非真伪二选一 | 需验证用户是否在敷衍、是否具备基础知识 |
| **苏格拉底法 / socratic-probe**（AVERT） | 通过**引导性提问**让用户自证，不预设错误结论 | 有明确可偏的市场「传闻/研报」作锚 | 用户防御型、需让其**主动解释**逻辑链 |
| **信息设计 / Bayesian 说服** | 优化披露以影响接收者**行动/信念** | 目标是**激发纠正**而非改变用户行为 | 需设计 persuasion 路径影响决策 |
| **信息操纵理论 / IMT** | 描述违反 Grice 准则的**操纵性沟通** | reactance 是 IMT 的一种**应用子集**（有偏陈述） | 分析 dodge、选择性披露等更广操纵 |
| **认知冲突 / 概念转变** | 制造「不对但说不清」的**认知失调** | 偏述指向**可快速识别**的事实性错误 | 需深度挖掘用户信念结构 |
| **SUE 价格对质** | 用**外部价格证据**与用户 claim 对质 | 尚无价格证据、不宜直接对质 | 已有 price_snapshot 且 claim 矛盾 |
| **cover-qa 掩护** | 维持互惠、低威胁 | resistance 低、可 CHALLENGE | resistance 高、需 RECOVER |
| **VA 可验证性追问** | 追 claim 中**缺失的可验证细节** | 用偏述**诱发**用户主动提供细节 | claim 已有框架，只缺参数 |

**一句话边界**：`reactance-biased-statement` = **用「可反驳的有偏传闻」换用户的主动纠正**；不是考卷（trap），不是纯提问（socratic），不是证据对质（SUE）。

## 6. 关键论文要点

### 6.1 Psychological Reactance Theory（Brehm, 1966; Brehm & Brehm, 1981）

- **核心命题**：自由是高度珍视的心理资源；威胁或消除自由会激发 motivational state（reactance），驱动**恢复自由**的行为。
- **操作化**：reactance 常表现为愤怒 + 负面认知（Dillard & Shen, 2005）；结果包括 boomerang effect、源贬损、行为反弹。
- **对本项目的启示**：有偏陈述应让用户感到「我可以选择不信/纠正」，而非「被强行说服」——纠正行为即 freedom restoration。

### 6.2 Freedom-prompting Reactance Mitigation Strategies（Communication Reports / Quarterly, 2021）

- **DOI**：10.1080/01463373.2021.1920443  
- **设计**：2（inoculation × 2 freedom-threatening language × 2 restoration postscript）× trait reactance，N=230，安全套宣传情境。
- **关键发现**：
  - 消息策略对 state reactance 的影响**因 trait reactance 而异**。
  - **高 trait reactance 者**：inoculation（预先说明可能感到自由威胁）** uniquely 降低** perceived threat to freedom，间接提升行为意向。
  - restoration postscript（结尾强调「选择权在你」）在不同 trait 水平下效果不一致。
- **对本项目的启示**：
  - 可对「高专业自尊 / 高 resistance 潜质」用户先做 **inoculation 式铺垫**：「市场上说法挺杂，我想听听您一线的实际感受」。
  - 偏述后可用 **restoration**：「当然最终以您那边情况为准，我只是想核实一下传闻。」
  - 需将 trait reactance / resistance_level 作为**路由分段变量**。

### 6.3 Psychological Reactance Theory and Politeness Theory（Communication Monographs, 2026）

- **DOI**：10.1080/03637751.2026.2615230  
- **问题**：威胁自由（PRT）与面子威胁（Politeness / facework）对说服抵抗的**竞争 vs 互补**解释。
- **关键发现**（综合同领域 2026 健康传播研究）：
  - PRT 与 Politeness **互补而非互斥**：控制性、面子威胁语言 → 自由威胁感知 ↑ → reactance ↑ → 态度/意向 ↓。
  - **mitigated face-threatening** 语言相对 aggravated 语言，显著降低 reactance 路径上的负面效应。
- **对本项目的启示**：
  - 有偏陈述的内容可以「错」，但**形式要礼貌**：间接引述、留余地、尊重用户专业权威（positive face）。
  - 避免同时触发「自由威胁 + 面子威胁」——例如「你们肯定不懂行情」类表述。

### 6.4 补充：Reactance 综述（Reynolds-Tylus, 2019, Frontiers in Communication）

- 降低 reactance 的 message features：减少 freedom-threatening language、提供选择权、共情叙事、two-sided message。
- **提高 reactance（本项目需适度利用）**：高控制语言、定论式框架、无选择余地的说服。
- **边界**：inoculation 在**高** freedom-threatening language 下可能失效——故本方向偏述强度应「可反驳」而非「高压说服」。

### 6.5 与 Readme 其他线的交叉引用

- **众包陷阱（WWW 2018）**：简化版——嵌入已知错误测真实性；reactance 版——嵌入**方向性偏误**测专业度与纠正意愿。
- **信息设计（Kamenica & Gentzkow; Kolotilin 2018）**：反向应用，从 rebuttal 反推 \(S\)。
- **IMT（Deceptively Dodging, 2018）**：有偏陈述 = 违反「量/质」准则的策略性披露，但需保持**可反驳**而非**不可检测的欺骗**。

## 附录：路由与 phase 建议

| 条件 | 建议 |
|------|------|
| resistance 低 + 有 partial claim + 缺量化 | `reactance-biased-statement` @ CHALLENGE |
| resistance 高 | RECOVER / `cover-qa`，禁用偏述 |
| 偏述后用户展开纠正 | 转 VERIFY / 轻量 VA 锁定细节 |
| 纠正内容与 price_snapshot 矛盾 | 下一轮交 `sue-price-confront` |
| 用户无纠正、仅「嗯/还行」 | 降级 `socratic-probe` 或换话题 |
