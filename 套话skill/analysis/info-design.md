# 信息设计 / Bayesian 说服 — 研究方向分析

> 研究来源：`optimal-info-disclosure.md`（Kolotilin, 2018）、`divide-and-inform-stub.md`（Accounting Review）、`Readme.md` §161–175  
> 应用场景：铁矿石电话客服套话（MarketTruthAgent 交互链路-1）  
> **核心思路**：策略性地选择「披露什么、隐瞒什么、分几批披露」，利用用户贝叶斯更新与专业纠偏本能，从反驳/补充中提取真实信息。

---

## 1. 核心机制

### 1.1 经典框架：发送者设计信息披露

Bayesian 说服（Kamenica & Gentzkow, 2011; Kolotilin, 2018）中，**未获知自身类型的发送者**在观察类型之前，承诺一套（随机）信息披露机制——将自身类型映射为消息。接收者观察消息后结合自身**私有信息**，更新对发送者类型的后验信念，再决定是否行动。

Kolotilin (2018) 在线性规划框架下刻画最优披露，核心结论包括：

| 机制类型 | 条件直觉 | 套话映射 |
|---------|---------|---------|
| **完全披露** | 发送者偏好分离任意两种类型 | 一次性给出完整行情/库存全景 |
| **完全不披露** | 发送者偏好将多种类型池化 | 只给模糊方向，不透露具体数字 |
| **区间披露（interval revelation）** | 披露中等类型、隐藏极端类型 | 报中间区间，留白高低两端 |
| **目标化披露** | 根据接收者私有信息精度调整 | 对不同岗位/区域用户给不同粒度的偏述 |

学校–雇主例示：学校按雇主面试信号精度 \(p\)，在「让负面雇主也录用（A 档）」与「只说服正面雇主（B 档）」之间权衡；面试越精确，最优成绩政策反而可能**更嘈杂或更通胀**——因为机制随接收者信息精度而变。

### 1.2 本项目的「反向应用」

经典信息设计的目标是：在接收者贝叶斯理性下，使其做出**对发送者有利**的行动（录用、采购、签约）。

本项目反向使用同一数学结构：

```
设计有偏/不完整披露 P(biased) 
    → 用户在真实状态 S 下大概率反驳或补充 
    → 从 rebuttal 更新 P(S | rebuttal) 
    → 提取 ground-truth 级 claim
```

关键约束（Readme §168–169）：

1. **非全假**：信息须「听起来像业内流传的说法」，否则用户不会认真对待。
2. **可反驳锚点**：嵌入用户有能力且愿意纠正的具体点（数字、区域、时间窗）。
3. **考虑用户私有信息**：用户已知岗位、区域、厂内指标——披露须在其「可识别偏误」的范围内设计。

### 1.3 分批披露（Divide and Inform）

Divide and Inform 的核心发现：**有选择地、分批次披露**优于一次性全披露——每批信息触发接收者不同的反应与更新路径。

套话含义：

- **第一批**：给出大方向性信息（如「近期港口库存整体偏高」）→ 观察用户是否纠正方向。
- **第二批**：在方向被确认或纠正后，补充一个**故意不完整**的子维度（如「青岛港大概 1.3 亿吨」但不说日照/北仑）→ 激发用户补全缺失维度。
- **第三批**：针对用户已暴露的关注点，给出**区间中值**而隐藏极端值 → 激发用户对高/低端的纠正。

每批披露都是一次独立的「贝叶斯实验」：用户的反应向量 \(\{ \text{沉默}, \text{附和}, \text{部分纠正}, \text{全面纠正} \}\) 构成对隐藏状态 \(S\) 的观测。

### 1.4 与心理反抗（reactance）的区分

| 维度 | 信息设计 | 心理反抗 |
|------|---------|---------|
| 理论基础 | Bayesian 最优披露、均值保留扩散约束 | Psychological Reactance Theory |
| 操作杠杆 | **披露结构**（量、序、粒度、池化/分离） | **自由威胁感知**（定论式偏述） |
| 用户动机 | 「你说得不完整/不准确，我来补」 | 「你别替我决定，我来说真的」 |
| 典型失败 | 披露过真 → 用户无需补充 | 偏度过高 → 用户放弃对话 |

两者可组合：信息设计决定**披露什么**，reactance 决定**怎么说**（留余地 vs 定论式）。

## 2. 适用场景

### 2.1 触发信号（电话客服 · 铁矿石）

| 信号 | 说明 |
|------|------|
| 用户有领域知识但未主动展开 | 岗位/口音/首问暗示其为贸易商、钢厂采购、港口物流等，却只说「了解一下」 |
| 存在可设计的「信息缺口」 | 公开行情有大致方向，但缺区域拆分、品位结构、交割细节——适合区间披露 |
| 多轮对话已建立基本信任 | RAPPORT 阶段完成，用户愿意接话，不宜首轮即高强度偏述 |
| 用户私有信息可推断 | 已知其区域（北方/南方港口）、品类偏好（粉矿/块矿），可针对性设计「在其知识范围内可识别」的偏误 |
| 需要结构化补全 partial claim | 用户已给出模糊表述（「库存不低」「价格还行」），缺量化与拆分 |

### 2.2 策略路由条件

- **interval revelation**：用户关注某一指标但尚未给区间；公开数据有宽分布。
- **rationed disclosure**：多个维度均未明确；需逐批试探用户最关心哪一维。
- **targeted pooling**：用户类型已知（如钢厂采购），将多种市场状态池化为一句模糊判断，等用户主动分离。
- **precision-adaptive**：用户前几轮表现出高专业度（纠正过公开数据）→ 下一批披露可更「精细地错」，因其私有信息更精确。

### 2.3 不适用信号

- 用户已给出完整、可验证的采购清单或数据口径。
- 用户明确「我不清楚」「没关注」——私有信息不足，无法识别偏误，无法触发有意义的纠正。
- 用户 resistance 极高、短答敷衍——应先 RECOVER / cover-qa，而非继续堆叠披露。
- 需要立即对质价格矛盾——应转 SUE，而非继续信息设计式采集。

## 3. 话术模板

### 模板 1：区间披露 — 只报中位，留白两端

- **触发条件**：讨论港口库存或价格，用户未给出区间；公开信息有宽分布。
- **机制对应**：Kolotilin 的 interval revelation——披露中等类型、隐藏极端类型。
- **示例话术**：
  > 「我们这边监测下来，主流港口铁矿库存大概在 **1.2–1.4 亿吨** 这个区间，整体中性偏宽松。您那边感受到的也差不多吗？」
- **预期反应**：
  - 专业用户：「不对，高点已经到 1.5 了」或「我们这边没那么高，1.1 左右」→ 提取真实区间端点。
  - 非专业用户：「嗯，差不多」→ 信号弱，换维度或降级 socratic-probe。

### 模板 2：分批披露 — 先方向后细节

- **触发条件**：多维度信息均未明确；首轮宜低威胁试探。
- **机制对应**：Divide and Inform 的 rationed disclosure。
- **示例话术（第一批）**：
  > 「最近进口矿市场，整体感觉是 **供应偏松、价格震荡偏弱**。您这边主要关心哪个方向？」
- **示例话术（第二批，用户选定「库存」后）**：
  > 「库存方面，北方港口好像比南方累得快一些。您方便说下是 **青岛/日照** 还是 **北仑/马迹山** 这边？」
- **预期反应**：用户在第二批纠正区域拆分或补充南方港口情况，暴露其业务锚点与真实感知。

### 模板 3：池化披露 — 模糊合并，等用户分离

- **触发条件**：用户未区分品类/产地，但市场实际存在明显分支；适合将多类型 pool 成一句。
- **机制对应**：Kolotilin 的 pooling vs separating——sender 池化两种类型，receiver 若掌握私有信息会倾向「分离」。
- **示例话术**：
  > 「澳洲主流矿和巴西矿最近走势差不多，都是 **稳中偏弱**。您有特别关注的矿种吗？」
- **预期反应**：
  - 专业贸易商：「不一样，卡粉坚挺，PB 粉松了」→ 分离品类，暴露其真实关注与头寸方向。
  - 泛泛用户：不纠正 → 信号弱。

### 模板 4：精度自适应 — 按用户专业度调整偏误粒度

- **触发条件**：用户前几轮已纠正过公开数据或使用过专业术语（如「62% 指数」「硅铝比」）。
- **机制对应**：Kolotilin 学校–雇主例中，接收者私有信息越精确，最优机制越「针对该类型」设计。
- **示例话术**：
  > 「62% 指数对标粉，港口现货大概 **820–840** 吧，主流成交集中在这段。您那边成交什么价位？」
- **预期反应**：高专业度用户纠正为「850 了」或「要看水分和港口」→ 提取更精细的真实成交区间与交割条件。

### 模板 5：故意遗漏维度 — 违反 Grice「量」准则

- **触发条件**：讨论需多参数（港口 + 品位 + 数量级）才能报价，用户只给了部分。
- **机制对应**：信息操纵理论中的「量」准则违反 + 信息设计的选择性披露。
- **示例话术**：
  > 「您说的这个品位，最近 **青岛港** 成交还算活跃。」（故意不说日照/北仑、不说湿吨/干吨、不说含税与否）
- **预期反应**：用户补充「我们走日照」「要算干基」「含税送到厂」→ 结构化补全 partial claim。

### 模板 6：时序分批 — 先旧后新，激发对「最新」的纠正

- **触发条件**：市场行情变化快，用户可能掌握更新信息。
- **机制对应**：Divide and Inform——不同批次触发不同更新。
- **示例话术（第一批）**：
  > 「上周港口库存还在累，大概日均增加 **50 万吨** 左右。」
- **示例话术（第二批）**：
  > 「不过这周好像有点变化，具体数据我还没更新。您那边有最新情况吗？」
- **预期反应**：用户纠正「这周已经开始去库了」或给出最新数字 → 提取时效性强的 ground truth。

### 模板 7：反向锚定 — 给出窄区间，激发用户扩宽

- **触发条件**：用户表述过窄（「就关心 PB 粉」），需探测其是否还关注其他标的。
- **机制对应**：与 interval revelation 对偶——隐藏「中间」、暴露宽尾，或反向操作。
- **示例话术**：
  > 「PB 粉最近确实比较紧，其他非主流矿好像都一般。您就盯 PB 吗？」
- **预期反应**：用户补充「金布巴也看」「巴西资源也询」→ 扩展需求图谱。

## 4. 风险与禁忌

### 4.1 主要风险

| 风险 | 表现 | 后果 |
|------|------|------|
| **过度披露** | 一次给出过多真实细节 | 用户无需纠正，采集失败 |
| **偏误不可识别** | 偏述超出用户知识范围 | 用户沉默或敷衍，无法区分「不专业」与「无信息」 |
| **分批过密** | 连续多轮偏述无正常业务交换 | 信任崩塌，用户警觉「你在套话」 |
| **忽略私有信息精度** | 对高专业用户给粗糙偏述 | 用户认为 agent 不专业，不予纠正 |
| **均值保留约束违反** | 偏述与已知公开数据严重矛盾 | 用户否定整段对话而非纠正单点 |
| **非单调效应** | 接收者信息越精确，最优披露反而更少 | 对「已经很懂」的用户继续堆叠偏述，边际收益为负 |

### 4.2 禁忌清单

1. **禁止**一次性全披露真实行情后再问「您那边呢」——用户没有纠正动机，违背信息设计初衷。
2. **禁止**编造与用户已知身份明显冲突的事实（如用户已知是北方港口贸易商，却只说南方且不做区域留白）。
3. **禁止**在用户已明确纠正后继续坚持原偏述——应从 rebuttal 中提取信息并转向 VERIFY，而非对抗。
4. **禁止**将用户纠正内容在当轮立即用于「算账式」对质——采集与对质分离，对质交给 `sue-price-confront`。
5. **禁止**在 RAPPORT 初期连续使用高强度区间偏述；至少 2–3 轮正常业务问答后再引入。
6. **禁止**忽视 Kolotilin 的非单调性：用户已展现高专业度时，应减少披露量、提高偏误精度，而非加大偏述幅度。

### 4.3 降级 / 退出条件

- 用户回应「不太清楚」「没关注」→ 偏述未命中专业点，换 `socratic-probe` 或 `cover-qa`。
- 用户仅「嗯/还行」无展开 → 本批披露无效，换维度或降级。
- 用户情绪升温、缩短回答 → 立即 RECOVER，暂停信息设计策略至少 3 轮。
- 用户主动追问「你数据哪来的」→ 轻量说明来源后转正常报价/答疑，暂停偏述链。

## 5. 与其他方向边界

| 对比方向 | 核心差异 | 何时选信息设计 | 何时选对方 |
|---------|---------|---------------|-----------|
| **心理反抗 / reactance** | 杠杆是自由威胁与纠正冲动 | 需设计**披露结构**（量、序、粒度） | 需用定论式偏述激发「别替我决定」 |
| **苏格拉底 / socratic-probe** | 引导性提问，不预设错误结论 | 有**可设计的披露对象**（行情、库存） | 用户防御型，需让其主动解释逻辑 |
| **陷阱问题 / trapping** | 有已知标准答案，测认真程度 | 无标准答案，靠**专业纠正** | 需验证用户是否敷衍、是否具备基础知识 |
| **信息寻求 / info-seeking** | 从用户**提问**反推隐藏状态 | 主动**披露**以激发用户反应 | 用户已在主动提问，倾听即可 |
| **SUE 价格对质** | 用外部证据与用户 claim 对质 | 尚无价格证据、不宜直接对质 | 已有 price_snapshot 且 claim 矛盾 |
| **隐式用户建模** | 从语言/行为估计 latent 需求 | 信息设计是**发送者侧编码** | 需理解用户已说了什么、推断意图 |
| **IMT / 信息操纵** | 描述违反 Grice 准则的操纵性沟通 | 信息设计是 IMT 的**形式化子集**（最优披露） | 分析 dodge、选择性回避等更广操纵 |

**一句话边界**：`info-design-disclosure` = **用「策略性不完整/有偏披露」的结构设计，换用户的贝叶斯更新与专业纠正**；不是考卷（trap），不是纯提问（socratic），不是心理对抗（reactance），不是证据对质（SUE）。

## 6. 论文要点

### 6.1 Kolotilin (2018) — Optimal Information Disclosure: A Linear Programming Approach

- **期刊**：Theoretical Economics 13 (2018), 607–635  
- **问题**：未获知类型的发送者设计披露机制；接收者拥有私有信息，在「行动/不行动」间选择。
- **方法**：线性规划刻画最优机制；单交叉（single-crossing）假设下，每条消息对应一个阈值类型。
- **核心结果**：
  - **完全披露最优** ⟺ 发送者偏好分离任意两种类型，而非池化。
  - **完全不披露最优** ⟺ 发送者偏好将三种类型池化，而非「池化两种 + 披露第三种」。
  - **区间披露最优**（在 sender 效用仅依赖期望类型时）：披露中等类型、隐藏极端类型；可行后验分布须满足**均值保留扩散**（mean-preserving spread）约束。
  - **比较静态**：发送者与接收者的期望效用**不必**随接收者私有信息精度单调变化——精度更高时，发送者可能选择披露更少信息；学校–雇主例中，面试越精确，成绩政策可能越通胀或越嘈杂。
- **对本项目的启示**：
  - 披露不是「越多越好」，而是**在用户私有信息精度与可识别偏误之间**优化。
  - 区间披露（报中位、藏两端）是理论上的最优结构之一，可直接映射为套话模板。
  - 分批披露时，每批应视用户已暴露的「私有信息精度」调整下一批的粒度与偏误幅度。

### 6.2 Divide and Inform — Rationing Information to Facilitate Persuasion

- **期刊**：The Accounting Review（DOI: 10.2308/accr-51707；PDF 未开放，stub 摘要）
- **核心发现**：当管理者与用户目标不一致时，**有选择地、分批次披露**优于一次性全披露；每批信息触发接收者不同的反应。
- **对本项目的启示**：
  - 多轮对话中，第一批给方向、第二批给细节、第三批给可纠正的窄锚点——符合 rationing 逻辑。
  - 不应在首轮即抛出完整「市场综述」，而应留待用户在每批反应中暴露其关注点后再定向披露。

### 6.3 Kamenica & Gentzkow (2011) — Bayesian Persuasion（理论基础）

- 发送者设计信息结构以影响接收者后验信念，进而影响行动。
- 凹化（concavification）方法：最优机制对应 sender 间接效用函数的凹包络。
- 本项目为**反向**：行动不是「录用/采购」，而是用户的**纠正/补充**；信念更新从 rebuttal 反推 \(P(S \mid \text{rebuttal})\)。

### 6.4 与 Readme 的交叉引用

- **§168–169**：策略本质是信息设计的反向应用——设计 \(P(\text{biased})\) 激发纠正，从 rebuttal 更新 \(P(S \mid \text{rebuttal})\)。
- **§174**：分批、有选择披露比一次全披露更有效——每批触发不同反应。
- **众包陷阱（§159）**：嵌入已知错误测真实性；信息设计是**方向性/结构性偏误**，无标准答案，靠专业纠正。
- **IMT（§176–186）**：有偏陈述 = 违反「量/质」准则的策略性披露；须保持可反驳，而非不可检测的欺骗。

| 用户已纠正过公开数据 | 下一批提高偏误精度（precision-adaptive） |
| 多维度均未明确 | rationed：先方向 → 后区域 → 后价位 |
| 用户无纠正、仅附和 | 换维度或降级 `socratic-probe` |
| 纠正内容与 price_snapshot 矛盾 | 下一轮交 `sue-price-confront` |
| resistance 高 | RECOVER / `cover-qa`，暂停信息设计链 |
