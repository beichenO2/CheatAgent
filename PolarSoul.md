# MarketTruthAgent — PolarSoul

## 项目定位

**一句话**：面向大宗商品（首期铁矿石）**情报客服**场景——解答客户行情咨询，同时在多轮对话中采集市场情报；分析链路结构化建模并评估回答真伪与来源可信度。

## 生态位置

- **独立研究/原型项目**，位于 `雷老师组测试任务` 工作区
- 可借鉴 Polarisor 生态中的 LLM 调用（PolarPrivate）、对话 Agent 模式（PolarClaw），但当前不硬依赖
- 产出：调研结论、Agent Benchmark 方案、cheatAgent 套话系统 + 分析后台（UI 暂缓）
- **当前阶段（2026-07-08）**：M6–M8 已完成；Alpha `alpha_v1` 7/10 用户 live（U001–U007），余下 mock

## 核心问题

Agent 与多个用户多轮对话后，收集到的市场微观信息（港口库存、采购积极性等）可能冲突、带激励偏差、存在同源传播。需要同时回答：

1. 哪个 claim 更可能是真的？
2. 哪个用户/来源更可靠？
3. 何时需要人工介入？

## 双链路架构（逻辑层）

```
┌─────────────────────────────────────────────────────────────┐
│                      交互链路（套话采集）                      │
│  用户输入 → UserModel → Skills路由 → cheatAgent套话 → 反馈      │
└──────────────────────────┬──────────────────────────────────┘
                           │ 对话记录（带时间戳）
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                      分析链路（真伪与信用）                    │
│  对话存储 → Claim抽取 → 欺骗检测 → 真值推断 → 外部校验 → 信用  │
└─────────────────────────────────────────────────────────────┘
```

详见 `decisions/002-dual-pipeline-architecture.md`。

## 数据边界

| 管什么 | 不管什么 |
|--------|----------|
| 用户-Agent 对话记录（文本+时间戳） | 用户手机号、真实身份 PII |
| 结构化 claim、来源可信度后验 | 完整交易/持仓明细 |
| 铁矿石领域本体与指标枚举 | 全品类大宗商品扩展（首期不做） |
| Benchmark 模拟数据与评测结果 | 生产级行情数据源接入（原型阶段用公开/模拟） |

## 接口约定（规划）

| 能力 | 协议/形态 | 状态 |
|------|-----------|------|
| 对话采集 API | REST / WebSocket | planned |
| Claim 存储 | JSON schema + SQLite/Postgres | planned |
| 分析任务 | 异步 job queue | planned |
| 演示 UI | Web 前端 | 暂缓（M5） |

## 关键约束（来自 Readme 假设）

- 用户知道对方是 Agent（非伪装人类）
- 场景限定铁矿石；已知用户头寸方向（如厂长≈多头）
- 电话客服约束：文本价值 >> 声线；用户问多答少，但互惠原则下「问二答一」
- 场外校验：价格不会骗人（延迟但可用）
